## B. 섹션별 수정 방향

### §1.3 성공 지표 — 보조 KPI 측정 경로 보강 (Q2, Q3 대응)

**문제:** 보조 KPI에 측정 도구·데이터 소스·분모 정의가 누락. 시점별 마일스톤 없음.
**수정:** 각 KPI에 측정 도구·분모 정의·Beta/GA 마일스톤 추가.

### §3 사용자 스토리 — 실패 케이스 AC 추가 (Q5~Q9 대응)

**문제:** 모든 Story에서 외부 API 장애, 룰 미등록, 데이터 미등록 등 실패/예외 케이스 AC 미정의.
**수정:** 각 Story에 실패 케이스 AC를 1~2개씩 추가 (AC*-F1, AC*-F2 등).

### §5 NFR — 누락 항목 보강 + 부하 테스트 기준 추가 (Q11~Q13 대응)

**문제:** 외부 API 타임아웃·재시도 정책, RPO/RTO, 부하 테스트 기준, 추가 모니터링 항목 누락.
**수정:** NFR 테이블에 행 추가, 모니터링 테이블에 외부 API 항목 추가.

### §4 기능 요구사항 — F5/F6 가치 근거 수치화 (Q18~Q19 대응)

**문제:** F5 "신뢰 적자 해소 필수"는 정성적 서술, F6 "통관 실패·CS 감소"는 목표 수치 없음.
**수정:** 경쟁사·JTBD 데이터 기반 수치 근거 추가.

### §8.2 실험 설계 — Before/After 통제 조건 보강 (Q16~Q17 대응)

**문제:** Before/After 설계에 관찰 기간·외생 변수 통제 미기술. "처리 속도 1.5배" 기준선 불명확.
**수정:** 관찰 기간·대조 조건·통계 검정 기준 추가.

### §9.3 차별 가치 — 산출 근거·측정 기준 명확화 (Q14~Q15 대응)

**문제:** "2배↑" 산출 근거 모호, "첫 구매 소요 시간" 시작·종료 기준점 미정의.
**수정:** 측정 기준(분자·분모), 시작·종료 이벤트 명시.
